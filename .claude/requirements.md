## サービスコンセプト

AIとの対話を通じて、旅の計画から「しおり」の作成・共有までをワンストップで完結する国内旅行向けWebサービス。

- **サービス名**: しおりっぷ (Shiorippu)
- **コンセプト**: 「しおり」+「trip」
- **ドメイン候補**: `shiorippu.com` or `shiori-trip.com`

## ターゲットユーザー

| 属性 | 詳細 |
|------|------|
| メイン | 家族旅行・友人グループ旅行の幹事 |
| サブ | カップル、少人数旅行 |
| 年齢層 | 20〜50代（スマホ・PC両方利用） |
| 利用シーン | 国内旅行の計画・準備・当日利用・思い出整理 |

## ユーザーフロー

```
STEP 0: ログイン（Google OAuth）
  ↓
STEP 1: AI旅行相談（選択肢カード型チャット → 7ステップヒアリング）
  ↓
STEP 2: しおりエディタ（AIの提案が反映された状態で編集スタート、D&D並び替え）
  ↓
STEP 3: デザイン・共有設定（テンプレート選択、招待リンク発行）
  ↓
STEP 4: プレビュー・出力（PDF出力、LINE共有、リンク共有）
```

## AI プロンプト設計

### ヒアリングフロー（7ステップ）

| Step | 質問 | 選択肢例 | 複数選択 |
|------|------|---------|---------|
| 0 | 行き先 | 北海道、京都・奈良、九州、沖縄 | × |
| 1 | 日程 | 1泊2日、2泊3日、3泊4日以上 | × |
| 2 | 人数・構成 | カップル、家族、友人グループ | × |
| 3 | テーマ | グルメ、温泉、観光、子供体験、自然 | ○ |
| 4 | 予算 | 〜5万、5〜10万、10〜20万、20万以上 | × |
| 5 | 出発地 | 東京駅、羽田/成田、車 | × |
| 6 | 特記事項 | アレルギー、バリアフリー、ペット、特になし | ○ |

### システムプロンプト概要

```
役割: 国内旅行のプロフェッショナルプランナー
ルール:
- 各質問は present_choices ツールで選択肢提示（3〜5個 + その他）
- 会話テキストは親しみやすく、絵文字を適度に使用
- 各スポットは search_spots / get_spot_details で裏取り
- プラン完成時は create_trip_plan で構造化出力
- 移動時間は calculate_route で正確に算出
```

### Function Calling ツール一覧

| ツール | 説明 |
|--------|------|
| `present_choices` | 選択肢カードを提示（question, step, choices[], multiSelect, allowOther） |
| `search_spots` | Google Places API でスポット検索（query, location, radius, type） |
| `get_spot_details` | スポット詳細取得（place_id → 住所, 営業時間, レビュー, 写真） |
| `create_trip_plan` | 旅行プランを構造化JSON生成（title, area, days[{events[]}]） |
| `calculate_route` | 移動ルート・所要時間計算（origin, destination, mode） |

> `present_choices` の詳細スキーマは要件定義 v3 セクション 8.2 を参照

## 非機能要件

### パフォーマンス目標

| 項目 | 目標値 |
|------|--------|
| 初回表示（LCP） | 2.5秒以下 |
| AI応答開始 | 1秒以下（ストリーミング） |
| ページ遷移 | 200ms以下（SPA遷移） |
| 同時編集 | 5人まで安定動作 |

### セキュリティ要件

- Supabase Auth + RLS
- 招待トークン: UUID v4（推測困難）
- HTTPS必須
- Server RoutesでAPIキー秘匿（Claude API, Google Maps）
- Rate Limiting（AI API呼び出し制限）
- XSS / CSRF対策

### 対応環境

- ブラウザ: Chrome, Safari, Firefox, Edge（最新2バージョン）
- モバイル: iOS Safari, Android Chrome
- レスポンシブ: 320px〜1920px

## リスクと対策

| リスク | 対策 |
|--------|------|
| AI応答品質（不正確なスポット情報） | Google Places APIで裏取り + 免責表示 |
| API利用料増大 | Rate Limiting + レスポンスキャッシュ |
| 共同編集の競合 | 楽観的ロック + Last Write Wins + 将来CRDT |
| LINE OAuth審査 | まずGoogleのみでMVP、後から追加 |
| Agent Teams間競合 | Lead定期統合 + 責任範囲分離 |
